Comandos más utilizados hasta el momento

  • git init: Permite crear un repositorio local
  • git status: Permite ver el estado de los archivos. Sin traza, agregados, modificados o eliminados
  • git add {nombre archivo}: Permite agregar archivos al staging area
  • git add .: Agrega todos los cambios de los archivos al staging area desde donde está posicionado la carpeta
  • git commit -m: Permite subir los archivos que están en el staging area a la rama actual (seleccionada)
  • git log --oneline: Muestra los commits, su descripción y donde está el puntero (HEAD)
  • git remote add origin {URL github}: Relaciona su repositorio remoto de destino desde el repositorio local
  • git remote -v: Permite visualizar si hay un repositorio remoto enlazado
  • git push origin {nombre branch}: Sube los cambios o los commits que tenga al repositorio remoto
  • git config user.name {username}: Configure el username
  • git config user.email {email}: Configure el email asociado a cuenta de github, hacia donde subirá los cambios
  • git config credential.username {username}: Configure credenciales de github con las que subirá los cambios del repositorio local hacia el repositorio remoto
  • git diff {#commit anterior} {#commit más reciente}: Muestra las diferencias cambios realizados en cada commit
  • git branch {nombre rama}: Permite crear un branch en el repositorio local
  • git checkout {nombre rama}: Permite seleccionar esa rama como rama activa de trabajo
  • git merge {nombre rama}: Selecciona esa rama para fusionar los cambios de esa rama a la rama activa
